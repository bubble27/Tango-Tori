# Tango Tori

Android app for Japanese learners. Paste or share a Japanese sentence; Tango Tori tokenizes it with Sudachi (furigana + POS color coding), looks each word up in a bundled JMdict + KANJIDIC database, and lets you turn any word into an AnkiDroid card with one tap.

Stack: Kotlin 2.0 · Jetpack Compose · Hilt · Room · Sudachi 0.7.5 · AnkiDroid public API.

---

## Project layout

```
app/                          Android app module
  src/main/java/com/tangotori/app/
    data/sudachi/             Sudachi wrapper + tokenizer
    data/db/                  Room entities + DAO for JMdict + KANJIDIC
    data/jmdict/              JMdict lookup repository
    data/anki/                AnkiDroid integration + card target repo
    data/IncomingSentenceBus  Bridge for shared sentences from Android share sheet
    domain/util/              Pure HTML / furigana / breakdown builders
    domain/usecases/          TokenizeSentence, LookupWord, CreateCard
    ui/sentence/              Sentence screen, ViewModel, expanded entry card
    ui/components/            FuriganaText, WordChip, TokenizedSentenceView
tools/jmdict-builder/         JVM Gradle subproject — builds the bundled DB
```

## Build

Standard Gradle Android setup. The toolchain expects JDK 17 and the Android SDK with `platforms;android-35` + `build-tools;35.0.0`.

```bash
# Debug build — fast, for iteration. Runs JIT-only so animations look slow,
# but feature correctness is fine.
./gradlew :app:installDebug

# Release build — R8-minified, signed with the debug keystore, AOT-compiled
# at install. Use for perf-sensitive testing (animations, scroll).
./gradlew :app:installRelease
adb shell cmd package compile -m speed -f com.tangotori.app
```

## Regenerating bundled data assets

Two large files are gitignored and need to be regenerated on a fresh clone before the APK will install correctly:

### 1. SudachiDict (`app/src/main/assets/sudachi/system_core.dic`, ~216 MB)

Download the **core** dictionary release zip from WorksApplications and unzip the `.dic` into `app/src/main/assets/sudachi/`:

```bash
curl -L -o /tmp/sudachi.zip \
  https://github.com/WorksApplications/SudachiDict/releases/download/v20230927/sudachi-dictionary-20230927-core.zip
unzip /tmp/sudachi.zip -d /tmp/
mv /tmp/sudachi-dictionary-20230927/system_core.dic app/src/main/assets/sudachi/
```

On Windows, OneDrive likes to dehydrate large bundled assets. After dropping the file in, `attrib +P` it so the cloud sync leaves it alone:

```bash
attrib +P app/src/main/assets/sudachi/system_core.dic
```

### 2. JMdict + KANJIDIC SQLite (`app/src/main/assets/jmdict.db`, ~61 MB)

Generated by the `:tools:jmdict-builder` subproject from scriptin/jmdict-simplified releases + jamsinclair/open-anki-jlpt-decks JLPT vocab. Run once:

```bash
./gradlew :app:kspDebugKotlin               # exports Room schema JSON
./gradlew :tools:jmdict-builder:run -Dorg.gradle.jvmargs="-Xmx2g"
```

The builder downloads `jmdict-eng-*.json.zip` and `kanjidic2-en-*.json.zip` from the latest scriptin release, plus the five JLPT vocab CSVs (set `JLPT_URL_PATTERN` env var to override the default URL).

After both assets are in place, run `:app:installDebug` or `:app:installRelease`.

## Feature flow

1. **Paste / share a sentence** — the sentence screen accepts text via the input field or via the Android share sheet (Tango Tori shows up as a share target for `text/plain`).
2. **Tokenization** — Sudachi splits the sentence; chips render inline with furigana above each kanji and a POS-color underline. Word list below mirrors the chips with dictionary forms + readings.
3. **JMdict lookup** — tapping a chip or word-list row expands it: numbered senses, POS, JLPT badge, Common badge, alternate kanji forms, multi-entry tabs for disambiguation.
4. **Anki card creation** — the expanded card has two inline buttons: *Add to default deck* (one tap once a default is set) and *Choose specific deck*. AnkiDroid's permission dialog appears on first use. Cards are added to a custom note type **"Tango Tori v5"** with fields for word, ruby furigana, meaning HTML, sentence HTML with target highlight, kanji breakdown, and tags (`tango-tori`, `common`, JLPT level, POS codes).

## Notes / gotchas

- This project lives under a OneDrive-synced path. The root `build.gradle.kts` redirects the Gradle build directory to `~/.tangotori-build/` so OneDrive doesn't dehydrate intermediate artifacts mid-build. Override with `TANGOTORI_BUILD_DIR`.
- Sudachi's `Tokenizer` is **not** thread-safe — `SudachiTokenizer.tokenize` serializes through a mutex so the share-intent path and the debounce path can't race on the lattice.
- The Anki note type version is bumped whenever the field set or template changes (currently v5). Old cards from earlier versions remain in AnkiDroid under their original model name; new submissions create the next version's model alongside.
- Debug-signed APKs run JIT-only on Android — animations and scrolling will look noticeably slower than release builds. Always verify perf with `:app:installRelease`.
